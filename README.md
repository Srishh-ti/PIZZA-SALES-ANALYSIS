## 🍕 Pizza Sales Data Analytics Project
# 🔍 Overview

This project analyzes pizza sales data to uncover business insights, improve decision-making, and understand customer purchasing patterns.

It demonstrates an end-to-end analytics workflow — starting from data storage in MySQL, followed by data cleaning, exploratory data analysis (EDA), and ending with a report and presentation created using Gamma.

# 📊 Dataset
- Domain: Food & Retail (Pizza Sales)
- Data Source: Pizza Sales Dataset
- Contents:
  - Order details
  - Pizza categories & sizes
  - Quantity and pricing
  - Date & time of orders
- Purpose:
  - To analyze sales performance, customer behavior, and revenue trends.

# 🛠️ Tools & Technologies
- Database: MySQL
- Query Language: SQL
- Analysis: SQL (for EDA & aggregation)
- Visualization & Reporting: Gamma (PPT)

# ⚙️ Project Steps
1. Data Loading (MySQL)
- Imported dataset into MySQL database
- Created structured tables for analysis
- Verified data types and schema
2. Data Cleaning
- Removed duplicates and inconsistent records
- Standardized column formats (dates, categories)
- Handled missing or incorrect values
- Ensured data integrity for analysis
3. Exploratory Data Analysis (EDA)
- Performed SQL-based analysis to answer key business questions:
  - Total revenue generated
  - Total number of orders
  - Best-selling pizzas
  - Revenue by category and size
  - Peak order times (day & hour)
  - Sales trends over time
4. Data Analysis & Insights
- Identified top-performing products
- Analyzed customer ordering behavior
- Evaluated sales contribution by category
- Discovered peak business hours and days
5. 📊 Dashboard / Presentation
- Created a structured presentation using Gamma
- Included:
  - KPI summaries (Revenue, Orders, Quantity)
  - Trend analysis (daily/hourly sales)
  - Top and bottom performers
  - Business insights

# 📌 Key Results
- Identified top-selling pizza categories and sizes
- Found peak order hours and busiest days
- Highlighted high-revenue products
- Discovered patterns in customer purchase behavior

# 🚀 How to Run
1. Setup MySQL Database
- Create a new database
- Import the dataset into MySQL tables
2. Run SQL Queries
- Execute SQL scripts to:
- Clean data
- Perform EDA
- Generate insights
3. Analyze Results
- Review query outputs
- Validate trends and metrics
4. View Presentation
- Open the Gamma PPT file to see:
- Visual insights

# 💡 Conclusion

This project showcases how SQL can be used to perform complete data analysis — from raw data to actionable insights. It highlights strong fundamentals in database handling, querying, and business-oriented analysis.
